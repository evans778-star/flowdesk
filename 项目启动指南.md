# Flowdesk 项目启动指南

本文档说明如何在本地启动 Flowdesk，以及如何按 dev/prod profile 配置运行环境。

> 不要把真实 DashScope API Key、JWT Secret、数据库密码、生产 IP、生产域名或本地绝对路径提交到 Git。

## 1. 环境要求

- JDK 21
- Maven Wrapper 已包含在仓库中；也可以安装 Maven 3.6+ 作为可选备用
- Docker Desktop
- DashScope API Key

推荐使用仓库自带的 Maven Wrapper；如果需要直接执行 `mvn`，请先安装 Maven 并配置 PATH。

## 2. 配置文件关系

项目使用以下配置文件：

```text
src/main/resources/application.yml
src/main/resources/application-dev.yml
src/main/resources/application-prod.yml
src/main/resources/application-example.yml
```

- `application.yml`: 通用配置。默认激活 `dev` profile，包含端口、上传、CORS、日志、AI 模型默认参数等。
- `application-dev.yml`: 本地开发配置，允许使用本地 MongoDB、Redis、默认管理员账号和本地 JWT Secret。
- `application-prod.yml`: 生产配置，敏感项必须通过环境变量提供。
- `application-example.yml`: 示例配置，用来查看配置项，不要填入真实密钥后提交。

## 3. 启动 MongoDB 和 Redis Stack

项目根目录提供 `docker-compose.yml`：

```powershell
docker compose up -d
docker compose ps
```

本地默认：

```text
MongoDB: 127.0.0.1:27017
Redis Stack: 127.0.0.1:6379
```

知识库检索依赖 Redis Stack / RediSearch。

## 4. 本地 dev 启动

PowerShell 设置环境变量：

```powershell
$env:SPRING_PROFILES_ACTIVE="dev"
$env:DASHSCOPE_API_KEY="your-dashscope-api-key"
$env:JWT_SECRET="replace-with-at-least-32-bytes-secret"
```

可选配置：

```powershell
$env:APP_CORS_ALLOWED_ORIGINS="http://localhost:3000,http://localhost:5173"
$env:UPLOAD_SAVE_PATH="upload/"
$env:UPLOAD_HOST="http://127.0.0.1:8888"
```

构建并启动：

```powershell
.\mvnw.cmd test
.\mvnw.cmd package
java -jar target/flowdesk-1.0.0-SNAPSHOT.jar
```

Linux/macOS:

```bash
./mvnw test
./mvnw package
java -jar target/flowdesk-1.0.0-SNAPSHOT.jar
```

已安装 Maven 时，也可以使用 `mvn test` 和 `mvn package` 作为备用。

默认访问：

```text
HTTP API: http://localhost:8888
WebSocket: ws://localhost:9000/ws
```

## 5. 生产 prod 启动

生产环境不要使用默认弱管理员凭据，不要使用本地 JWT 默认值。至少配置：

```powershell
$env:SPRING_PROFILES_ACTIVE="prod"
$env:DASHSCOPE_API_KEY="your-production-dashscope-api-key"
$env:JWT_SECRET="replace-with-strong-production-secret"
$env:FLOWDESK_ADMIN_USER="your-admin-user"
$env:FLOWDESK_ADMIN_PASSWORD="your-strong-admin-password"
$env:MONGODB_URI="mongodb://user:password@host:27017/Flowdesk?authSource=admin"
$env:REDIS_HOST="your-redis-host"
$env:REDIS_PORT="6379"
$env:REDIS_PASSWORD="your-redis-password"
$env:REDIS_DATABASE="0"
$env:APP_CORS_ALLOWED_ORIGINS="https://your-frontend.example.com"
$env:UPLOAD_SAVE_PATH="/data/flowdesk/upload/"
$env:UPLOAD_HOST="https://your-api.example.com"
```

上面的值是占位示例，不要直接提交真实生产值。

## 6. 关键配置项

### CORS

```yaml
app:
  cors:
    allowed-origins: ${APP_CORS_ALLOWED_ORIGINS:http://localhost:3000,http://localhost:5173}
```

生产环境必须设置为真实前端域名，不要使用通配来源。

### 文件上传

```yaml
upload:
  save-path: ${UPLOAD_SAVE_PATH:upload/}
  host: ${UPLOAD_HOST:http://127.0.0.1:8888}
  max-file-size-bytes: ${UPLOAD_MAX_FILE_SIZE_BYTES:52428800}
  allowed-extensions: ${UPLOAD_ALLOWED_EXTENSIONS:.pdf,.txt,.md,.csv,.doc,.docx,.xls,.xlsx,.ppt,.pptx,.png,.jpg,.jpeg,.gif,.webp}
  allowed-content-types: ${UPLOAD_ALLOWED_CONTENT_TYPES:application/pdf,text/plain,text/markdown,text/csv,application/msword,application/vnd.openxmlformats-officedocument.wordprocessingml.document,application/vnd.ms-excel,application/vnd.openxmlformats-officedocument.spreadsheetml.sheet,application/vnd.ms-powerpoint,application/vnd.openxmlformats-officedocument.presentationml.presentation,image/png,image/jpeg,image/gif,image/webp}
```

`upload.save-path` 对应环境变量 `UPLOAD_SAVE_PATH`。

### DashScope

```powershell
$env:DASHSCOPE_API_KEY="your-dashscope-api-key"
```

### JWT

```powershell
$env:JWT_SECRET="replace-with-at-least-32-bytes-secret"
```

## 7. 接口鉴权

- `/v1/user/login` 可以匿名访问。
- `/ws/**` 可以建立连接，WebSocket 握手层会读取 token。
- 其他 HTTP 接口默认需要 JWT。
- `/api/debug/memory/**` 只在 `dev` profile 下注册。
- `/api/knowledge/diag/**` 只在 `dev` profile 下注册。
- `/actuator/**` 未默认公开。

## 8. 常见问题

### Maven 命令不存在

优先使用仓库自带的 Maven Wrapper：Windows 执行 `.\mvnw.cmd test`，Linux/macOS 执行 `./mvnw test`。如需直接使用 `mvn`，请安装 Maven 并配置 PATH。

### MongoDB 连接失败

检查：

```powershell
docker compose ps
```

确认 MongoDB 容器运行，端口为 `27017`。

### Redis / 知识库检索失败

确认 Redis Stack 容器运行，端口为 `6379`，并支持 RediSearch。

### AI 调用失败

检查 `DASHSCOPE_API_KEY` 是否设置、Key 是否有效、账号是否有模型调用权限。

### 上传失败

检查：

- `UPLOAD_SAVE_PATH` 是否存在或可创建
- 目录是否可写
- 文件大小是否超过 `UPLOAD_MAX_FILE_SIZE_BYTES`
- 扩展名是否在 `UPLOAD_ALLOWED_EXTENSIONS`
- MIME 类型是否在 `UPLOAD_ALLOWED_CONTENT_TYPES`
